完整问题清单：搜索 → 播放 → 歌单链路

代码验证 + 用户反馈 + RN 对比

搜索页问题（4项）

S1

搜索仅触发当前 Tab 的单一插件源

当前: SearchViewModel.search() 只调用 _selectedPlugin 的 plugin.search()

RN 原版: 搜索同时触发所有可搜索插件，结果按二级 Tab（插件名）分组展示

结构差异: RN 用两层 Tab — 第一层媒体类型（单曲/专辑/歌手/歌单），第二层插件源。Android 只有一层插件 Tab + SegmentedButton

S2

搜索页顶部 Bar 高度异常

当前: 使用 Material3 TopAppBar（标题 "搜索" + 返回箭头）+ 独立的 OutlinedTextField

RN 原版: 自定义 NavBar rpx(88) 高，内嵌药丸形搜索输入框 rpx(64) 高 + 右侧"搜索"按钮。没有单独标题栏

修复: 去掉 TopAppBar，改用自定义 Row 内嵌搜索框，对齐 RN 布局

S3

页面过渡动画未对齐原版

当前: 全局统一 slideIntoContainer 100ms 水平滑动

RN 原版: 使用 React Navigation 默认过渡（iOS 风格右滑入/出）

需确认: 是动画方向还是时长不对？还是需要特定页面使用不同动画？

S4

搜索结果无「添加到歌单」入口

当前: SearchResultItem 只有 clickable（点击播放），无长按/三点菜单

RN 原版: 每条结果右侧有三点菜单 ellipsis-vertical，点击弹出选项面板含"添加到歌单"

复用: LocalScreen 已有 AddToPlaylistDialog 可复用

播放器问题（3项）

P1

无封面高斯模糊背景

当前: 纯色背景

RN 原版:

  • 底层: 纯黑 #000 背景
  • 上层: 封面图 blurRadius=50, opacity=0.5
  • 两层叠加 = 暗色模糊效果

Android 实现: Coil + Modifier.blur(50.dp) 或 RenderScript/RenderEffect

P2

顶部 Bar 歌曲/歌手信息未居中

当前: 使用 Material3 TopAppBar，title 默认左对齐

RN 原版: NavBar 高 rpx(150)，中间区域 flex:1 + alignItems:"center" + justifyContent:"center"，标题和歌手严格居中

结构: 左返回箭头 | 居中(标题+歌手+平台tag) | 右分享按钮

P3

封面图可能不显示（数据链路待验证）

代码分析: artwork 字段从 JS 插件 → JsBridge → MusicItem → PlayerState → PlayerScreen 链路完整

JsBridge 支持字段: artwork, coverimg, cover, pic, img, image, avatar 等 20+ 别名

可能原因:

  • 订阅源插件 JS 代码返回的字段不在已知别名列表中
  • 插件返回的 URL 格式有问题（需运行时日志确认）
  • Coil 网络请求被 Referer/UA 拦截

建议: 安装订阅源后实际运行，抓日志确认 artwork 值
