搜索 → 播放 → 添加到歌单：链路断裂分析

基于代码验证的功能完整性报告

链路总览

Search → Play → Add to Playlist Flow
搜索页 → 插件解析URL → ⚠ 播放(URL可能为空) → 导航到播放器

✗ MiniPlayer封面未加载 ✗ 播放器无高斯模糊 ✗ 搜索结果无「加入歌单」

功能断裂点

1

MiniPlayer 封面图未加载

文件: MiniPlayerContent.kt

coverUri 字段已从 PlayerState 正确映射，但 UI 只显示 Icons.Default.MusicNote 占位符，从未使用 AsyncImage 加载实际封面。PlayerScreen 中有正确实现可参考。

2

搜索结果无「添加到歌单」入口

文件: SearchScreen.kt → SearchResultItem

搜索结果只有 clickable（点击播放），没有长按、三点菜单或任何添加到歌单的入口。
对比: LocalScreen 已有长按 → AddToPlaylistDialog 的完整实现可复用。RN 原版使用三点菜单 + 底部面板。

3

播放器无封面高斯模糊背景

文件: PlayerScreen.kt

当前使用纯色背景。RN 原版在 musicDetail/components/background.tsx 中使用 Image blurRadius={50} + opacity 0.5 作为全屏背景。Android 需要用 Compose 的 Modifier.blur() 或 RenderEffect 实现。

4

首页 HomeRoute 不显示 MiniPlayer

文件: MainActivity.kt:61

showRealMiniPlayer = !isHomeRoute && !isPlayerRoute
首页使用 mock MiniPlayer（测试数据），真实 MiniPlayer 不在首页展示。需要统一为真实 MiniPlayer。

潜在风险点

A

URL 为空时静默失败

文件: SearchViewModel.kt:245, MusicItemMediaExt.kt:12

如果插件 getMediaSource() 返回空 URL，fallback 回退逻辑可能仍传递 blank URL 给 ExoPlayer。toMediaItem() 中 url?.let { setUri(it) } 在 url 为 null 时静默跳过，ExoPlayer 无法播放。

B

MediaController 连接静默失败

文件: PlayerController.kt:250-287

withConnectedController 在 controller 为 null 且连接失败时静默丢弃操作，用户无反馈。

  • ✅ MiniPlayer → PlayerScreen 导航
  • ✅ PlayerScreen 封面图加载（AsyncImage）
