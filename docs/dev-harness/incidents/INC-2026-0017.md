## INC-2026-0017 — 纯秒小数 LRC 时间戳 [s.ff] 未识别导致歌词退化为 plain-text fallback

### 根因

`LyricParser.parseTimestampMs` 用 `tag.split(':')` 后强制 `parts.size in 2..3`，`timestampTokenRegex` 与 `LyricDocument.lyricTimestampRegex` 也都要求至少一个 `\d+:` 前缀，所以 `[265.35]` 这种 RN `timeReg=/\[[\d:.]+\]/g` 兼容的纯秒小数时间戳全被拒。timed line 个数变 0，整段 LRC 退化成 plain-text fallback，`isTimed=false`、所有 `timeMs=0`。

### 复发条件

任一处单独把时间戳 regex 收紧成 `\\[(?:\\d+:)+\\d+...]`、把 `parseTimestampMs` 的 `parts.size` 收回 `2..3`，或新增 `LyricDocument` 之外第 N 处独立 regex 与上面两处不一致——contract test 用 `[5.5]/[10]/[mm:ss.xx]/[hh:mm:ss.xx]` 黄金样本调用 `LyricParser.parse`，立即触发断言。

### 教训

LRC 时间戳格式必须按 RN `timeReg` 三档对齐（`[hh:mm:ss(.ff)?]`、`[mm:ss(.ff)?]`、`[s(.ff)?]`），且 parser、token strip、document 推导三处 regex 必须保持同一份字面量。任何放宽/收紧都得跑 `*LyricParserTest`、`*LyricTimingTest` 与本 contract test。
