## INC-2026-0023 — APK 安装入口长期依赖 FileProvider+Intent 跨进程授权，2 次修复未收敛 → 迁移到 PackageInstaller.Session

- id: INC-2026-0023
- area: updater
- date: 2026-05-22
- status: active
- rule_ref: —
- guard:
    type: manual
- fix_ref: 见 commits `fdaec87c`、`4bb368b5`、`ca77cea7`（`fix-installer-session` 分支）；设计 spec [`docs/superpowers/specs/2026-05-22-apk-install-package-installer-migration-design.md`](../../superpowers/specs/2026-05-22-apk-install-package-installer-migration-design.md)

### 根因

APK 安装入口持续使用 `content://` URI 跨进程传给厂商安装器，FileProvider 临时授权在 ROM 安装器包装层内不稳定丢失：

- 第一轮（2026-05-19，commit `96a79737`）：补 `ClipData` + 显式 `grantUriPermission`，无效。
- 第二轮（2026-05-20，commit `1b5f5c0b` / `6c462908`）：改 `ACTION_VIEW` → `ACTION_INSTALL_PACKAGE`，以本地 emulator 作为唯一验收证据，无效。

两轮修复均只在 Intent 标志位上打补丁，未触及根本：`Intent.ACTION_INSTALL_PACKAGE` 自 API 25 deprecated，架构上应换用 `PackageInstaller.Session`，从源头消除跨进程 URI 授权链路。

### 复发条件

在 `:updater` 的安装入口（`ApkInstaller` / 任何新增安装入口）中重新引入 `ACTION_VIEW` 或 `ACTION_INSTALL_PACKAGE`，并依赖 FileProvider `content://` URI 跨进程传给厂商安装器。

### 教训

- 连续 2 次针对同一症状的局部修复未收敛，是"质疑架构选型"的信号，不应继续在同一层面叠加补丁。
- 以本地 emulator 解析通过作为唯一验收证据，无法覆盖 MIUI / OPPO / 三星等厂商 ROM 的安装器包装行为；真机验收是安装链路的强制要求。
- `PackageInstaller.Session` 是 Android 官方推荐的第三方应用安装入口，自 API 21 可用，不应因"看起来复杂"而延迟迁移。
