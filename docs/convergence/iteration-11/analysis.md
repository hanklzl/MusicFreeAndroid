# 迭代 11 分析（搜索可用但点击播放失败）

## 背景
- 上一轮已修复插件扩展字段丢失问题（`songmid` 透传）。
- 但真实场景仍可能存在：当前选中插件 `getMediaSource` 临时失败，导致“能搜到但点不了”。

## 问题本质
- 搜索页点击播放仅尝试一次当前插件解析。
- 若该插件接口波动、风控或返回结构变化，即直接失败，无兜底路径。

## 本轮目标
- 在 `SearchViewModel.resolveAndPlay()` 增加兜底：
  - 先走当前选中插件；
  - 失败时自动回退到 `元力WY` 搜索同曲并解析播放地址。
- 引入可测试的匹配策略，避免回退命中随机歌曲。
